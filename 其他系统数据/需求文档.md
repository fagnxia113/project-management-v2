## 1. 项目概述

### 1.1 项目背景

企业内部项目数量不断增加，涉及的人员和资产资源需要在不同项目之间灵活调配。前期采用的低代码/配置驱动方案在应对复杂审批流转换、多维度成本核算及高频资源调拨时，由于抽象度过高，导致扩展极其和维护成本巨大。因此，本项目决定回归**标准自建系统架构**。

### 1.2 项目目标

构建一个**高性能、高扩展性**的Web项目管理系统，其核心特征是从“配置驱动”转向“模型驱动+代码实现”，确保业务逻辑的直观性和灵活性。

**核心目标：**
- **回归标准开发**：采用 SpringBoot/Go + React/Vue 标准技术栈，用成熟的后端逻辑代替复杂的配置引擎。
- **精细化资产管理**：实施“一物一码”原则，确保每一台仪器/假负载的物理流转可追溯。
- **无限极任务体系**：采用路径枚举（Path Enumeration）实现 WBS 任务无限分级，支撑精准进度卷叠。
- **工时效能统计**：日报升级为 Timesheet 工时单，实现人工费用的项目分摊与效能统计。
- **深度移动端/企微集成**：核心审批流直接对接企业微信审批 API，实现“PC填单、手机审批”。

### 1.3 设计理念

#### 1.3.1 架构设计 (Standard MVC/Service Architecture)

```
┌─────────────────────────────────────────┐
│  表现层 (React + Ant Design/Tailwind)    │
│  - 标准组件，逻辑直观，高度响应式              │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  业务逻辑层 (Standard Services)          │
│  - 核心逻辑：EquipmentService, TaskService  │
│  - 审批中心：对接企业微信 API               │
└─────────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────┐
│  数据持久层 (MySQL/PostgreSQL)           │
│  - 强一致性关系模型                      │
│  - 一物一码台账、WBS 路径树               │
└─────────────────────────────────────────┘
```

#### 1.3.2 优化核心优势

| 维度 | 旧版 (简道云/JSON配置版) | 优化版 (标准自建版) |
|------|---------|------------|
| 资产管理 | 聚合统计，难追溯具体单品 | **一物一码**，全寿命周期追踪 |
| 任务级数 | 固定层级，计算开销大 | **无限层级**，路径枚举助力高效汇总 |
| 扩展成本 | 修改引擎逻辑，极高风险 | 修改代码业务逻辑，风险可控、标准明确 |
| 移动集成 | 依赖平台 App，体验受限 | **原生集成企业微信**，体验流畅 |
| 审批流程 | 自建重度引擎，维护难 | **外包给企微审批**，轻量化状态机管理 |

### 1.4 项目范围（优化调整）

**包含功能：**
- **项目管理模块**：支持 WBS 无限嵌套、进度自动卷叠、甘特图展示。
- **人事模块**：组织架构与企微同步、入职/离职记录。
- **仪器/资产模块**：型号定义 -> 物理实例管理 -> 条码流转。
- **工时/日报模块**：日报挂载任务工时、自动分摊项目人力成本。
- **审批/预警中心**：对接企业微信审批流、逾期/未填消息推送。

**不再包含功能：**
- 自定义的 JSON 表单/流程引擎配置（改为标准代码开发）。
- 本地复杂的 BPMN 流程编辑器（改为企微审批）。

### 1.5 核心数据模型 (Core Optimized Data Models)

- **设备型号字典 (EquipmentModel)** & **物理实例 (EquipmentInstance)**：分离定义与实体。
- **项目任务 (ProjectTask)**：增加 `wbs_path` 字段，实现树形结构。
- **工时细目 (TimesheetEntry)**：关联日报与任务，精细化记录人天。
- **流转单据 (TransferOrders)**：配合状态机记录每一次资产/人力的流转轨迹。

## 4. 业务流程

### 4.1 人员相关流程

#### 4.1.1 人员入职流程

```
[部门提交入职申请]
      ↓
[人事部门审批]
      ↓
[人力资源审批] ←-- [拒绝：结束]
      ↓
[审批通过]
      ↓
[系统自动执行：
 - 人员信息写入人事人员管理表
 - 初始化权限（基础查看权限）
 - 管理员配置具体权限]
```

#### 4.1.2 人员离职流程

```
[部门提交离职申请]
      ↓
[人事部门审批]
      ↓
[部门经理审批] ←-- [拒绝：结束]
      ↓
[审批通过]
      ↓
[系统自动执行：
 - 人员信息写入人事人员管理表（标记离职）
 - 管理员撤销/修改权限
 - 释放占用的项目资源]
```

#### 4.1.3 行政调动流程

```
[申请人提交调动申请（部门/分公司）]
      ↓
[原部门经理审批]
      ↓
[目标部门经理审批]
      ↓
[人事部门审批] ←-- [拒绝：结束]
      ↓
[审批通过]
      ↓
[系统自动执行：
 - 人员信息写入人事人员管理表
 - 部门归属更新
 - 管理员同步更新权限]
```

#### 4.1.4 人员跨项目调拨流程

```
[项目经理提交申请]
      ↓
[系统检测资源冲突]
      ↓
[系统提示冲突并给出建议] ←-- [无冲突：可直接提交]
      ↓
[提交审批]
      ↓
[部门经理审批]
      ↓
[人力资源审批] ←-- [拒绝：结束]
      ↓
[审批通过]
      ↓
[系统自动更新：
 - 人员状态改为"占用"
 - 添加到目标项目成员
 - 更新人员日历]
```

```
[仪器状态恢复为对应状态]
```

#### 4.2.7 仪器报废流程

```
[提交报废申请]
      ↓
[填写报废信息：报废原因、报废日期]
      ↓
[资产管理员审批]
      ↓
[财务部门审批] ←-- [拒绝：结束]
      ↓
[审批通过]
      ↓
[系统自动执行：
 - 流程信息写入仪器报废管理表
 - 仪器状态自动调整为"报废"
 - 同步更新对应出库管理表]
```

### 4.3 项目管理流程

#### 4.3.1 任务接收确认流程

```
[项目经理创建任务并分配]
      ↓
[实施人员收到任务通知]
      ↓
[实施人员确认接收]
      ↓
[系统自动执行：
 - 将流程信息写入任务管理表单
 - 任务状态更新为"进行中"]
```

#### 4.3.2 日报提交流程

```
[实施人员进入日报提交页面]
      ↓
[填写日报内容：任务进展、完成情况、工时记录]
      ↓
[上传附件（可选）]
      ↓
[提交日报审核]
      ↓
[项目经理审核] ←-- [驳回：要求修改]
      ↓
[审批通过]
      ↓
[系统自动执行：
 - 日报数据存入报表
 - 关联任务进度自动更新]
```

#### 4.3.3 日报未提交提醒流程

```
[系统每日定时检查（如22:00）]
      ↓
[筛选今日未提交日报的人员]
      ↓
[发送平台内消息提醒]
      ↓
[同时发送邮件提醒]
      ↓
[更新日报统计看板数据]
```

#### 4.3.4 子任务评价提交流程

```
[子任务完成后，实施人员标记完成]
      ↓
[管理人员收到评价通知]
      ↓
[填写评价信息：
   - 完成任务数量
   - 完成质量评分
   - 实际人天投入
   - 评语]
      ↓
[提交评价]
      ↓
[评价数据存入子任务评价管理表]
      ↓
[统计分析自动更新]
```

#### 4.3.5 项目结项流程

```
[项目经理提交项目结项申请]
      ↓
[上传结项材料：
   - 项目总结报告
   - 成果交付物
   - 财务结算单
   - 人员/仪器结项清单]
      ↓
[部门经理审批]
      ↓
[项目管理办公室审批]
      ↓
[财务部门审批] ←-- [拒绝：修改后重新提交]
      ↓
[审批通过]
      ↓
[系统自动执行：
 - 流程信息写入项目结项管理表
 - 更新项目所有任务状态为"已结项"]
      ↓
[项目归档]
```

#### 4.3.6 里程碑预警流程

```
[系统每日定时检查]
      ↓
[筛选条件：计划日期 - 当前日期 ≤ N天 且 未完成]
      ↓
[生成预警列表]
      ↓
[查找对应项目经理]
      ↓
[检查邮件配置]
      ↓
[发送邮件提醒]
      ↓
[标记里程碑为"已提醒"]
```

## 10. 数据驱动架构扩展指南

### 10.1 新增实体操作步骤

假设要新增一个"会议室"实体：

#### 步骤1：更新实体元数据 (EntityMeta.json)

```json
{
  "entities": {
    "ConferenceRoom": {
      "name": "会议室",
      "fields": {
        "id": { "type": "string", "primaryKey": true },
        "name": { "type": "string", "required": true, "label": "会议室名称" },
        "capacity": { "type": "number", "label": "容量" },
        "location": { "type": "string", "label": "位置" },
        "facilities": { "type": "array", "itemType": "string", "label": "设施" },
        "status": {
          "type": "enum",
          "enumRef": "RoomStatus",
          "label": "状态",
          "default": "available"
        }
      }
    }
  }
}
```

#### 步骤2：更新枚举配置 (EnumConfig.json)

```json
{
  "enums": {
    "RoomStatus": {
      "label": "会议室状态",
      "values": [
        { "value": "available", "label": "可用", "color": "green" },
        { "value": "occupied", "label": "占用中", "color": "red" },
        { "value": "maintenance", "label": "维护中", "color": "orange" }
      ]
    }
  }
}
```

#### 步骤3：更新表单配置 (FormConfig.json)

```json
{
  "forms": {
    "ConferenceRoomCreate": {
      "entity": "ConferenceRoom",
      "mode": "create",
      "title": "新建会议室",
      "sections": [/* ... */]
    }
  }
}
```

#### 步骤4：更新页面配置 (PageConfig.json)

```json
{
  "pages": {
    "ConferenceRoomList": {
      "route": "/conference-rooms",
      "title": "会议室管理",
      "component": "DataGridPage",
      "dataSource": "ConferenceRoom",
      "config": { /* ... */ }
    }
  }
}
```

#### 步骤5：更新菜单配置 (MenuConfig.json)

```json
{
  "key": "conference-room",
  "label": "会议室管理",
  "route": "/conference-rooms"
}
```

**完成！** 系统自动识别配置并渲染，无需修改任何代码。

### 10.2 修改审批流程操作步骤

假设要新增一个"会议室预约"审批流程：

#### 步骤1：更新工作流配置 (WorkflowConfig.json)

```json
{
  "workflows": {
    "ConferenceRoomBooking": {
      "name": "会议室预约流程",
      "type": "approval",
      "variants": [{
        "key": "booking",
        "label": "预约审批",
        "nodes": [/* 定义流程节点 */]
      }]
    }
  }
}
```

#### 步骤2：更新权限配置 (PermissionConfig.json)

```json
{
  "roles": {
    "user": {
      "permissions": ["conference_room:view,booking_create"]
    }
  }
}
```

**完成！** 流程自动生效。
