# 简道云项目管理系统 - 完整设计方案（V6.0）

## 一、需求分析

### 1.1 功能需求清单

**项目管理模块：**
1. 任务管理：项目经理创建任务（里程碑、子任务），支持任务分配流程
2. 进度汇报：实施人员通过日报提交进度，支持定时提醒
3. 日报大屏：展示已提交/未提交人员情况
4. 子任务评价：管理人员评价实施人员完成情况
5. 项目人员信息管理：查看项目人员情况（在岗、请假）
6. 项目结项申请：提交结项信息，更新项目状态
7. 任务信息大屏：甘特图展示项目进展
8. 项目进度大屏：甘特图展示项目阶段进度

**设备管理模块：**
1. 设备管理：假负载、仪器信息管理，支持状态维护（正常、轻微损坏、影响使用、维修中、报废）
2. 设备入库管理：维修入库、项目结束入库、采购入库
3. 设备出库管理：维修出库、报废出库、项目领用
4. 设备调拨A（国内有库存）：仓库发货
5. 设备调拨B（国内无库存）：外借或采购后发货
6. 设备调拨C（国外项目间）：双方项目经理审批
7. 设备维修流程：更新设备状态为维修中
8. 设备报废流程：更新设备状态为报废

**人事管理模块：**
1. 组织架构搭建：机构分组、人员信息录入
2. 人员入职：入职流程，添加权限
3. 人员离职：离职流程，回收权限
4. 人员调动：部门/分公司调动，更新权限

### 1.2 核心设计原则

1. **主表存储基础数据**：项目、项目阶段、设备、员工、客户、仓库
2. **流程表单记录业务**：入职、离职、调动、入库、出库、调拨、维修、报废、任务分配、日报等
3. **中间表记录关联关系**：项目人员
4. **统计信息实时计算**：在仪表盘中通过统计组件实时计算展示，避免使用聚合表
5. **工作流条件分支**：数据更新规则通过工作流条件分支实现条件判断

**进度计算公式：**

```
计划进度 = 
  IF(阶段状态="已完成", 100,
    IF(阶段状态="未开始", 0,
      IF(预计使用天数<=0, 0,
        MIN(100, ROUND(已使用天数 / 预计使用天数 * 100, 0))
      )
    )
  )

进度偏差 = 计划进度 - 完成进度

预警状态 = 
  IF(阶段状态="已完成" OR 阶段状态="未开始", "正常",
    IF(进度偏差 >= 10, "严重滞后",
      IF(进度偏差 >= 5, "预警", "正常")
    )
  )
```

**预警逻辑说明：**

| 预警状态 | 触发条件 | 说明 |
|---------|---------|------|
| 正常 | 进度偏差 < 5% | 进度正常 |
| 预警 | 5% ≤ 进度偏差 < 10% | 进度落后，需要关注 |
| 严重滞后 | 进度偏差 ≥ 10% | 进度严重落后，需要立即处理 |

| 备注 | 多行文本 | - | - | - |

**负责人自动确定规则：**
- 位置状态="在库" → 负责人 = 关联仓库.仓库负责人（通过数据联动获取）
- 位置状态="在项目" → 负责人 = 项目名称.项目经理（通过数据联动获取）
- 位置状态="维修中"或"调拨中" → 负责人保持不变

**简道云实现方式：**
由于简道云数据联动不支持条件判断，负责人字段需要在流程中自动更新：
1. 设备入库单审批通过 → 负责人 = 入库仓库.仓库负责人
2. 设备调拨单收货确认 → 负责人根据调入位置自动更新
3. 设备出库单审批通过 → 负责人 = 目标项目.项目经理

**当前位置公式：**
```
IF(位置状态="在库", 关联仓库.仓库名称, 
   IF(位置状态="在项目", 项目名称.项目名称,
      IF(位置状态="维修中", "维修中", "调拨中")))
```

**说明：** 简道云公式支持通过点号引用关联数据字段，如"关联仓库.仓库名称"。如果遇到问题，可改用数据联动或手动填写。

**三个状态字段说明：**

| 字段 | 含义 | 选项 | 变化场景 |
|-----|------|------|---------|
| 主机状态 | 设备完好程度 | 正常/轻微损坏/影响使用/维修中/报废 | 维修、报废流程 |

**设计优势：**
1. 合并仪器和假负载，减少表数量

4. 采用"一物一码"模式，便于追踪设备全生命周期
5. 负责人自动确定，简化管理
6. 新增配件情况、详图等字段，信息更完整

**阶段自动结项逻辑（简道云实现说明）：**

**方式1：仪表盘提醒**
- 在"项目进度大屏"中添加组件：各阶段任务完成情况统计
- 显示：阶段名称、总任务数、已完成任务数、未完成任务数
- 当阶段下所有任务完成时，阶段负责人手动提交阶段更新单

**方式2：定时提醒**
- 配置定时提醒：每天检查各阶段任务完成情况
- 当某阶段所有任务状态="已完成"时，发送消息通知阶段负责人
- 阶段负责人提交项目阶段更新单，更新阶段状态为"已完成"

**方式3：手动触发**
- 在任务结项申请审批通过后，手动检查该阶段其他任务是否全部完成
- 如全部完成，提示"该阶段所有任务已完成，是否更新阶段状态？"
- 点击确认后，自动更新项目阶段表.阶段状态="已完成"

**建议采用方式2**，通过定时提醒通知阶段负责人，由负责人确认后更新阶段状态，确保流程可控。

### 7.1 智能助手Pro概述

**功能说明：** 智能助手Pro是简道云的自动化工作流工具，支持定时触发、表单触发、HTTP触发等多种触发方式，可实现复杂的业务自动化。

**支持的节点类型：**

|---------|------|

| 执行节点 | 新增数据、修改数据、删除数据、查询单条数据、查询多条数据 |
| 控制节点 | 条件分支、循环容器 |
| 计算节点 | 数据计算 |
| 插件节点 | 发送邮件、生成打印文件、打开数据新增页等 |

### 7.2 进度偏差预警（智能助手Pro）

**功能说明：** 每天定时检查项目阶段进度，当进度偏差达到预警阈值时，自动发送邮件通知相关负责人。

**智能助手Pro配置：**

| 配置项 | 设置值 |
|-------|--------|
| 智能助手名称 | 进度预警通知 |
| 触发方式 | 定时触发 |
| 触发时间 | 每天 09:00 |

**工作流设计：**

```
【触发节点】定时触发（每天 09:00）
    ↓
【查询节点】查询项目阶段表
    │ 过滤条件：
    │   - 阶段状态 = "进行中"
    │   - 预警状态 = "预警" OR "严重滞后"
    ↓
【循环容器】遍历查询结果
    │
    ├─→ 【条件分支】判断预警类型
    │       │
    │       ├─→ 预警 → 发送预警邮件
    │       └─→ 严重滞后 → 发送严重滞后邮件
    │
    └─→ 【插件节点】发送邮件
            │ 收件人：阶段负责人、关联项目.项目经理
            │ 主题：【项目预警】{项目名称}-{阶段名称}
```

**配置步骤：**

1. 进入应用后台 → 智能助手Pro
2. 点击「新建智能助手」
3. 命名为「进度预警通知」
4. 添加「定时触发」节点，设置触发时间为每天 09:00
5. 添加「查询多条数据」节点，选择表单：项目阶段表，设置过滤条件
6. 添加「循环容器」节点，选择循环数据为上一步查询结果
7. 在循环容器内添加「插件节点」，选择「发送邮件」
8. 设置收件人：循环项.阶段负责人、循环项.关联项目.项目经理
9. 设置邮件主题和正文
10. 保存并启用智能助手

### 9.1 权限配置原则
