# 项目管理模块 (Project Management) - 详细设计

## 1. 业务概述
项目管理模块是系统的核心入口，负责从项目立项、WBS 拆解、任务执行到项目结项的全生命周期管理。

## 2. 核心功能深入解析

### 2.1 WBS (工作分解结构) 设计
系统通过 `WBSService` 维护任务的层级编号逻辑。
- **编号规则**: 采用点分十进制（如 `1.1.2`）。顶级节点为里程碑，后续节点根据父节点自动生成序列。
- **自动重计算**: 当任务顺序或归属发生变化时，系统支持 `recalculateWBS`，确保所有编号保持逻辑连贯。
- **层级关系**: 支持无限层级，但在前端展示上通常优化为 3-5 层以保持清晰。

### 2.2 任务管理实现
- **任务模型**: 存储在 `tasks` 表，包含计划开始/结束时间、实际时间、进度百分比、权重。
- **关联逻辑**: 任务与设备（调拨）、资产、日报评价等强关联。
- **任务状态机**: `draft` -> `assigned` -> `in_progress` -> `review` -> `completed`。

### 2.3 项目阶段与变更
- **阶段流转**: 每个项目可自定义 `ProjectPhase`。
- **变更控制**: 所有的关键变更（如延期、预算调整）必须走工作流审批。

## 3. 数据库模型 (Prisma)
- **`projects`**: 存储项目主体信息、客户关联、负责人（PM/Lead）。
- **`tasks`**: 核心任务表。
- **`project_phases`**: 阶段定义。
- **`task_attachments`**: 任务交付物文档存储记录。

## 4. API 规范 (部分)
- `GET /api/project/wbs/:projectId`: 获取整个项目的 WBS 树。
- `POST /api/project/task/rollback`: 任务进度回滚申请。
- `PUT /api/project/update_status`: 配合审批流自动更新项目状态。

## 5. 前端核心组件
- **`ProjectGantt`**: 甘特图视图。
- **`WBSManager`**: 拖拽式的 WBS 编辑器。
- **`WorkLogList`**: 关联显示该项目的所有日报及任务反馈。
