# 汇升智合项目管理系统 (v2) - 系统与功能设计文档

## 1. 引言

本系统是一个基于元数据驱动的企业级项目管理系统，旨在提供高度灵活、可配置的功能模块，涵盖项目生命周期、设备管理、人员组织、审批流程及数据统计等核心业务。

## 2. 技术栈架构

### 2.1 后端架构
- **运行环境**: Node.js 18+
- **Web 框架**: Express
- **数据库 ORM**: Prisma (配合 MySQL 8.0)
- **核心逻辑**: TypeScript 实现，强调类型安全与模块化。
- **第三方集成**: 企业微信 (WeCom) 用于用户同步与消息提醒。

### 2.2 前端架构
- **框架**: React 18
- **构建工具**: Vite
- **样式**: Tailwind CSS
- **路由**: React Router v7
- **状态管理/数据获取**: TanStack Query (React Query)
- **表单引擎**: Formily (高扩展性表单解决方案)
- **图标库**: Lucide React

---

## 3. 系统核心设计理念

### 3.1 元数据驱动 (Metadata-driven)
系统的核心灵活性源于元数据配置。
- **EntityMeta.json**: 定义所有业务实体的字段、类型、关联关系及验证规则。
- **EnumConfig.json**: 定义系统范围内的枚举值，如项目状态、审批结果等。
- **核心价值**: 通过修改 JSON 配置文件，即可在前端自动渲染列表/过滤，在后端自动生成通用 CRUD API。

### 3.2 统一表单服务 (Unified Form Service)
- **模块化**: 集成了项目、设备、人员、任务等模块的表单。
- **动态渲染**: 基于 Formily 渲染引擎，支持字段级显隐控制、联动验证、异步选项加载。
- **解耦**: 表单定义与业务代码解耦，支持版本管理与草稿保存。

## 4. 核心功能板块设计

### 4.1 项目管理模块 (Project Management)
- **全生命周期**: 从立项（Proposal）到进行中（In Progress），直至完成（Completed）。
- **WBS 管理**: 支持工作分解结构，任务父子关系追踪。
- **分阶段控制**: 自定义项目阶段，根据阶段触发不同的流程。
- **多维度视图**: 列表视图、看板视图，以及统计报表。

### 4.2 设备管理模块 (Equipment Management)
- **一物一码**: 每一台设备拥有独立的管理编码（Manage Code）和序列号。
- **配件体系**: 支持主设备与配件的关联管理。
- **状态流转**: 闲置、使用中、维修中、报废、已调拨。
- **全链条操作**: 
  - **入库管理**: 支持采购入库、调拨入库。
  - **调拨流程**: 跨项目/仓库的设备移动，带有发货、收货确认环节。
  - **维修/报废**: 完整的在线申请与审批记录。

### 4.3 组织人事模块 (Organization & Personnel)
- **多级部门**: 支持从企业微信同步多级部门树，维护部门负责人信息。
- **全生命周期管理**: 
  - **入职流程 (Onboard)**: 线上申请、信息采集、OA 账号同步。
  - **异动/转岗 (Transfer)**: 记录员工在不同项目或部门间的变动。
  - **离职流程 (Offboard)**: 资产交接监控、权限停用。
- **员工档案**: 记录基本信息、技能标签、出勤成本、当前状态（在岗/休假/单飞等）。

### 4.4 增强型工作流引擎 (Workflow Engine)
- **BPMN 兼容**: 内部实现了类 BPMN 的节点逻辑（开始、结束、用户任务、网关）。
- **流程网关 (Gateway)**: 
  - **排他网关 (Exclusive)**: 基于业务变量（如金额、地区）自动路由分支。
  - **并行网关 (Parallel)**: 同时触发多个子流程或任务。
- **动态审批人**: 根据业务上下文自动解析，支持“部门负责人”、“项目经理”、“特定角色”等。
- **多样化审批模式**:
  - **或签**: 一人通过即可。
  - **会签**: 全体通过才可进入下一环节。
  - **投票**: 达到设定的比例即视为通过。
- **事件联动**: 支持在流程特定阶段触发代码执行（例如审批结束后自动更新设备台账）。

### 4.5 智能提醒与自动化 (Intelligence & Automation)
- **采购提醒 (Purchase Reminder)**: 基于物料库存量和项目需求自动触发采购预警。
- **日报提醒**: 在规定时间内对未提交日报的员工进行系统或企微通知。
- **进度分析**: 自动汇总各项目 WBS 任务完成情况。

### 4.5 考勤与日报 (Attendance & Daily Reports)
- **考勤统计**: 结合项目地点的考勤异常监控。
- **日报系统**: 每日工作内容摘要、任务进度反馈。
- **任务评价**: 针对完成任务进行质量反馈。

## 5. 通用接口规范

### 5.1 元数据 API
- `GET /api/metadata/entities`: 获取全部实体定义。
- `GET /api/metadata/enums`: 获取系统字典。

### 5.2 通用 CRUD
- `GET /api/data/:entity`: 支持分页、排序、过滤。
- `POST /api/data/:entity`: 自动适配元数据定义的创建。
- `PUT /api/data/:entity/:id`: 部分或全部更新。

---

## 6. 数据库设计要点

- **关系严密性**: 依赖 Prisma 维护强类型外键关系。
- **扩展性**: 大量使用 `JSON` 类型存储非结构化配置（如流程节点配置、表单自定义配置）。
- **审计追踪**: 所有关键表均包含 `created_at` 和 `updated_at`，审批订单记录完整的审计日志（Audit Logs）。
